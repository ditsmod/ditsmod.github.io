Що таке Ditsmod

Ознайомлення з Ditsmod​

Ditsmod є Node.js веб-фреймворком, його назва складається із DI + TS + Mod, щоб підкреслити важливі складові: він має Dependency Injection, написаний на TypeScript, та спроектований для хорошої Модульності.

Головні особливості Ditsmod:

  • Модульна архітектура на декораторах, що дозволяє декларативно описувати структуру застосунку.
  • Зручний механізм указання та вирішення залежностей між різними класами: ви в конструкторі указуєте інстанси яких класів вам потрібні, а DI бере на себе непросту задачу "як їх отримати".
  • Можливість написання власних розширень (інколи їх називають плагінами), що можуть асинхронно ініціалізуватись, і що можуть залежати один від одного.
  • Можливість динамічно додавати та видаляти модулі після старту вебсервера, без необхідності рестарту.
  • Має підтримку OpenAPI.
  • На сьогодішній день, Ditsmod є одним із найшвидших серед Node.js веб фреймворків.

Деякі концепції архітектури Ditsmod взяті із Angular концепцій. Модуль DI фактично витягнутий з Angular v4.4.7. (з мінімальними допрацюваннями) та інтегрований в Ditsmod.

Встановлення​

Мінімальний базовий набір для роботи застосунку має репозиторій ditsmod-seed. Клонуйте його та встановіть залежності:

git clone https://github.com/ditsmod/seed.git my-app
cd my-app
yarn

Запуск​

yarn start

Таку команду можна використовувати для розробки застосунку, оскільки після кожного збереження вашого коду, ви зразу бачите ці зміни.

Перевірити роботу сервера можна за допомогою curl:

curl -isS localhost:3000

Або просто перейшовши у браузері на http://localhost:3000/

Компіляція застосунку та запуск сервера в продуктовому режимі відбувається за допомогою команди:

yarn build
yarn start-prod

Більше прикладів застосунку є у теці examples, а також у репозиторії RealWorld.

Вхідний файл для Node.js​

Після встановлення Ditsmod seed, перше, що необхідно знати: весь код застосунку знаходиться у теці src, він компілюється за допомогою TypeScript-утиліти tsc, після компіляції попадає у теку dist, і далі вже у вигляді JavaScript-коду його можна виконувати у Node.js.

Давайте розглянемо файл src/main.ts:

import 'reflect-metadata';
import { Application } from '@ditsmod/core';

import { AppModule } from './app/app.module';

new Application().bootstrap(AppModule).catch((err) => {
  console.log(err);
});

Після компіляції, він перетворюється на dist/main.js та стає вхідною точкою для запуску застосунку, і саме тому ви будете його вказувати у якості аргументу для Node.js:

node dist/main.js

Варто звернути увагу на import 'reflect-metadata' у першому рядку файла. Цей модуль необхідний для роботи декораторів у Ditsmod, але його достатньо указувати єдиний раз у вхідному файлі для Node.js.

Такий імпорт потрібно робити і для тестів, оскільки при тестуванні вхідним файлом вже буде файл тесту, а не dist/main.js. Наприклад, якщо ви будете використовувати jest у якості фреймворку для тестів, а файл test-file.js міститиме скомпільований тест, то щоб запустити його ось так:

jest test-file.js

у даному файлі повинен бути імпорт reflect-metadata.

Проглядаючи далі файл src/main.ts, ви можете бачити, що створюється інстанс класу Application, а у якості аргументу для методу bootstrap() передається AppModule. Тут AppModule є кореневим модулем, до якого вже підв'язуються інші модулі застосунку.
